# SlipQ Patient Journey 분석 프로젝트

> 불면증 환자 여정 분석 & 넛지 전략 도출

---

## 📌 프로젝트 개요

**목표:**
1. 불면증 환자들의 Patient Journey 분석
2. 어떤 넛지를 어디서 걸 수 있을지 도출
3. 보고용 슬라이드 1부 작성

**기간:** 2025.01.27 ~ 2025.02.07

**담당:** Coree + 인턴

---

## 📊 현황

| 항목 | 상태 |
|------|------|
| 내부 데이터 | ~150건 (요청 예정) |
| 외부 데이터 | 수집 필요 |
| 핵심 타겟 | 의사 처방 뚫기 |

---

## 🔄 Phase 정리

### Phase 0: 프레임 세팅 (1/27~1/28)

| 할 일 | 산출물 | 상태 |
|-------|--------|------|
| 0-1. Patient Journey 단계 정의 | Journey 프레임워크 문서 | ⬜ |
| 0-2. 각 단계별 수집할 데이터/소스 정의 | 데이터 소스 매핑표 | ⬜ |
| 0-3. 분석 도구(대시보드) 기본 설계 | 대시보드 와이어프레임 | ⬜ |

**Journey 프레임:**
```
[인지] → [정보탐색] → [자가치료] → [병원고민] → [병원선택] → [내원] → [처방]
```

---

### Phase 1: 메타 분석 (1/29~1/31)

> **목표:** "불면증 환자가 병원까지 어떻게 오는지" 큰 그림 그리기

| 할 일 | 데이터 소스 | 산출물 | 상태 |
|-------|-------------|--------|------|
| 1-1. 불면증 진료/처방 현황 분석 | 심평원 공개 데이터 | 처방 규모, 진료과별 분포 | ⬜ |
| 1-2. 불면증 검색 트렌드 분석 | 네이버 Lab API | 검색량 추이, 연관 키워드 | ⬜ |
| 1-3. 환자 커뮤니티 탐색 | 카페/블로그/유튜브 | 주요 커뮤니티 리스트, 논의 주제 | ⬜ |
| 1-4. 기존 연구/리포트 리뷰 | 학회지, 시장 리포트 | 기존 Journey 가설 정리 | ⬜ |

**Phase 1 산출물:** Patient Journey 가설 맵 (v0.1)

---

### Phase 2: 데이터 수집 & Journey 검증 (2/1~2/3)

> **목표:** 가설 Journey를 데이터로 검증 & 구체화

| 할 일 | 데이터 소스 | 산출물 | 상태 |
|-------|-------------|--------|------|
| 2-1. 환자 커뮤니티 크롤링 | 카페/블로그 | 단계별 고민/행동 패턴 | ⬜ |
| 2-2. 내부 데이터 분석 | 슬립큐 150건 | 실제 환자 경로, 유입 채널 | ⬜ |
| 2-3. 전문가 인터뷰 (가능 시) | 의사/약사 | Journey 가설 검증 | ⬜ |
| 2-4. Journey 맵 업데이트 | 위 종합 | 검증된 Journey 맵 (v1.0) | ⬜ |

**Phase 2 산출물:** 검증된 Patient Journey 맵 + Back Data

### Phase 3: 터치포인트 & 넛지 분석 (2/4~2/5)

> **목표:** Journey 단계별 "어디서 어떤 넛지 걸 수 있는지" 도출

| 할 일 | 데이터 소스 | 산출물 | 상태 |
|-------|-------------|--------|------|
| 3-1. Journey 단계별 터치포인트 정리 | Phase 2 결과 | 터치포인트 맵 | ⬜ |
| 3-2. SNS/광고 경향 분석 | 인스타/유튜브/네이버 | 경쟁사 벤치마크, 광고 패턴 | ⬜ |
| 3-3. 소비 패턴 분석 | 커뮤니티, 쇼핑 트렌드 | 불면증 관련 소비 행동 | ⬜ |
| 3-4. 넛지 옵션 리스트업 | 위 종합 | 단계별 넛지 옵션표 | ⬜ |
| 3-5. 우선순위 평가 | 도달률/비용/실행가능성 | 넛지 우선순위 매트릭스 | ⬜ |

**Phase 3 산출물:** 터치포인트별 넛지 전략표
